# 扫雷关键内存偏移记录

> **适用版本**：Windows 7 经典扫雷 `winmine.exe`（x86）
>
> **查找工具**：Cheat Engine 7.x

## 核心偏移

| 地址 | 类型 | 含义 |
|------|------|------|
| `0x01005361` | `BYTE[24][32]` | 雷区数据起始地址，每格 1 字节 |
| `0x01005334` | `DWORD` | 雷区宽度（列数） |
| `0x01005338` | `DWORD` | 雷区高度（行数） |
| `0x01005194` | `DWORD` | 基址测试标记（初级模式） |

## 雷区数据编码

| 字节值 | 含义 |
|--------|------|
| `0x0F` | 未打开的格子 |
| `0x10` | 行结束标记 |
| `0x41` | 数字 1 |
| `0x42` | 数字 2 |
| `0x43` | 数字 3 |
| `0x44` | 数字 4 |
| `0x45` | 数字 5 |
| `0x46` | 数字 6 |
| `0x47` | 数字 7 |
| `0x48` | 数字 8 |
| `0x8F` | 雷 |
| `0xCC` | 已标记的格子 |

## 扫雷窗口消息

| 消息 | wParam | 含义 |
|------|--------|------|
| `WM_COMMAND` | `0x209` | 启动"初级"游戏 |
| `WM_COMMAND` | `0x20A` | 启动"中级"游戏 |
| `WM_LBUTTONDOWN` | `MK_LBUTTON` | 鼠标左键按下 |
| `WM_LBUTTONUP` | `0` | 鼠标左键释放 |

## 格子坐标换算

```
X = startX + col * 16 + 8   // 16 = 格子边长, 8 = 半格居中
Y = startY + row * 16 + 8
```

其中 `startX=15`, `startY=60` 为雷区左上角相对窗口的偏移（需根据实际分辨率校准）。

## 查找偏移的方法

1. 打开 Cheat Engine，附加到 `winmine.exe`
2. 首次扫描：新游戏 → 扫描 `0`；点开一个格子 → 扫描该格子的数值（如 `1` 即扫描 `0x41`）
3. 反复操作直至锁定 1-3 个地址
4. 使用 "Find out what writes to this address" 定位基址
5. 记录基址偏移到本文件

## 注意事项

- 偏移值**依赖具体扫雷程序版本**（Win7 vs Win10 vs 第三方版本）
- 不同难度下雷区基址可能相同，但尺寸偏移（`0x5334` / `0x5338`）不变
- `ReadProcessMemory` 需要以**管理员权限**运行程序
- 雷区数据每行 32 字节，但实际只使用前 W 列；行尾以 `0x10` 标记
